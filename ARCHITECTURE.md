# Goon 架构文档

## 项目概述

Goon 是一个基于 Cobra 和 Gin 的 Go Web 项目生成器，用于快速初始化和管理模块化的 Web 项目。

## 架构设计

### 分层架构

```
┌─────────────────────────────────────┐
│         CLI 层 (cmd/)               │
│  - 命令定义和参数解析                │
│  - 用户交互和输出                    │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│      生成器层 (internal/generator/)  │
│  - 项目初始化                        │
│  - 模块生成                          │
│  - 路由注册                          │
│  - 代码重构                          │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│      模板层 (internal/template/)     │
│  - 模板渲染                          │
│  - 自定义模板支持                    │
│  - 模板函数                          │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│      工具层 (internal/utils/)        │
│  - 命名转换                          │
│  - 备份管理                          │
│  - 安全验证                          │
└─────────────────────────────────────┘
```

### 核心组件

#### 1. CLI 层 (cmd/)

**职责**：
- 定义命令行接口
- 解析用户输入
- 调用生成器层执行任务
- 格式化输出结果

**主要命令**：
- `init`: 初始化新项目
- `add`: 添加新模块
- `remove`: 删除模块
- `list`: 列出所有模块
- `rename`: 重命名模块
- `generate`: 生成代码（测试等）

#### 2. 生成器层 (internal/generator/)

**职责**：
- 执行代码生成逻辑
- 管理文件系统操作
- 处理模块依赖关系
- 实现回滚机制

**核心文件**：
- `project.go`: 项目初始化
- `module.go`: 模块生成
- `router.go`: 路由注册
- `rename.go`: 模块重命名
- `test.go`: 测试生成
- `pkg.go`: 功能包管理

#### 3. 模板层 (internal/template/)

**职责**：
- 管理模板文件
- 渲染模板内容
- 支持自定义模板
- 提供模板函数

**模板类型**：
- 项目模板：初始化项目时使用
- 模块模板：生成模块时使用
- 测试模板：生成测试文件时使用

#### 4. 工具层 (internal/utils/)

**职责**：
- 提供通用工具函数
- 命名格式转换
- 文件备份和恢复
- 安全性验证

**核心功能**：
- 命名转换（snake_case, camelCase, PascalCase, kebab-case）
- 备份管理（支持回滚）
- 路径验证（防止路径遍历）
- 输入清理（防止注入攻击）

## 数据流

### 添加模块流程

```
用户输入
  │
  ▼
CLI 解析参数
  │
  ▼
生成器验证输入
  │
  ▼
创建备份
  │
  ▼
渲染模板
  │
  ▼
写入文件
  │
  ▼
注册路由（可选）
  │
  ▼
返回结果
```

### 错误处理流程

```
操作失败
  │
  ▼
检查备份
  │
  ▼
执行回滚
  │
  ▼
恢复原状态
  │
  ▼
返回错误信息
```

## 设计模式

### 1. 模板模式
使用 Go 的 `text/template` 进行代码生成，支持自定义模板覆盖内置模板。

### 2. 策略模式
不同的命名转换策略（snake_case, camelCase 等）可以灵活切换。

### 3. 备忘录模式
使用 BackupManager 保存文件状态，支持回滚操作。

### 4. 工厂模式
NewRenderer(), NewHandler(), NewService() 等工厂函数创建对象。

## 扩展性

### 添加新命令

1. 在 `cmd/` 目录创建新的命令文件
2. 实现命令逻辑
3. 在 `init()` 函数中注册到 rootCmd

### 添加新模板

1. 在 `internal/template/templates/` 目录添加模板文件
2. 模板会自动被加载
3. 或者使用自定义模板目录

### 添加新功能包

1. 在 `internal/generator/pkg.go` 的 `availablePackages` 中添加映射
2. 创建对应的模板文件
3. 功能包会自动可用

## 性能优化

### 1. 模板缓存
模板在初始化时一次性加载，避免重复解析。

### 2. 并行处理
批量操作时可以并行处理多个模块（未来优化）。

### 3. 增量更新
只更新变化的文件，跳过已存在的文件（幂等性）。

## 安全性

### 1. 路径验证
使用 `ValidatePath` 防止路径遍历攻击。

### 2. 输入清理
使用 `SanitizeInput` 清理用户输入，防止注入攻击。

### 3. 备份机制
所有文件操作前先备份，失败时自动回滚。

## 测试策略

### 单元测试
- 每个核心功能都有对应的测试文件
- 使用 `testing` 包和 `testify` 断言库
- 测试覆盖率目标：80%+

### 集成测试
- 测试完整的命令执行流程
- 使用临时目录进行测试
- 验证生成的文件内容

### 测试文件位置
- `*_test.go` 文件与被测试文件在同一目录
- 使用 `t.TempDir()` 创建临时测试环境

## 配置管理

### 配置文件
- `.goonrc.yaml`: 项目配置文件
- 支持自定义模板路径
- 支持默认层配置
- 支持命名风格配置

### 配置优先级
1. 命令行参数
2. 项目配置文件
3. 用户主目录配置文件
4. 默认配置
