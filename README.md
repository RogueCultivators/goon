# Goon - Go Web 项目生成器

> **⚠️ 重要提示：本代码完全由 AI 生成**
>
> This code is completely generated by AI

基于 Cobra 和 Gin 的命令行工具，用于快速初始化和管理模块化的 Web 项目。

## 特性

- 🚀 快速初始化 Gin Web 项目
- 📦 模块化架构设计（按模块分组）
- 🎨 基于 Go template 的代码生成
- 🔧 内置常用中间件（CORS、Logger、限流、认证、权限控制等）
- 🐳 Docker 和 Docker Compose 支持
- 📊 集成 SQLC 支持
- 📝 统一的响应格式
- 🗂️ 清晰的项目结构
- ⚡ 支持 minimal 模式，只生成核心文件
- 🎯 灵活的层级生成，按需选择要生成的代码层
- 🔄 自动路由注册，无需手动修改路由文件
- 🔤 智能命名转换，支持驼峰、下划线等多种命名格式
- 🔁 幂等性支持，可安全重复执行命令

## 使用方法

### 初始化新项目

```bash
# 完整模式（包含所有功能包）
goon init myproject

# Minimal 模式（只生成核心文件）
goon init myproject --minimal

# 指定 Go module 名称
goon init myproject -m github.com/username/myproject
```

可选参数：
- `-m, --module`: 指定 Go module 名称（默认使用项目名称）
- `--minimal`: 只生成核心文件和目录（响应、日志、错误处理等）

**幂等性支持**：可以安全地重复运行 `goon init` 命令，已存在的文件会被跳过，不会被覆盖。

### 添加新模块

```bash
cd myproject

# 生成完整模块（handler, service, model, repository, schema）
goon add user

# 只生成指定的层
goon add product --layers=handler,service

# 生成模块但不自动注册路由
goon add order --register=false

# 支持多种命名格式（自动转换）
goon add userProfile      # 生成 user_profile 目录
goon add user-management  # 生成 user_management 目录
```

可选参数：
- `-l, --layers`: 指定要生成的层，可选值：handler, service, model, repository, schema
- `--register`: 是否自动在 router.go 中注册路由（默认为 true）

生成的文件位于 `internal/[module_name]/` 目录：
- `handler.go` - HTTP 处理器
- `service.go` - 业务逻辑层
- `model.go` - 数据模型
- `repository.go` - 数据访问层
- `schema.go` - 请求/响应结构体

**幂等性支持**：可以安全地重复运行 `goon add` 命令，已存在的文件会被跳过，不会被覆盖。这对于补充生成缺失的层非常有用。

**自动路由注册**：默认情况下，`goon add` 会自动在 `internal/router/router.go` 中注册模块路由，包括标准的 CRUD 端点。

### 自定义模板

1. 在 `internal/template/templates/` 目录下创建或修改 `.tmpl` 文件
2. 使用 Go template 语法：`{{.VariableName}}`
3. 模板会通过 `embed.FS` 自动嵌入到二进制文件中

示例模板：
```go

    service *Service
}

func NewHandler(service *Service) *Handler {
    return &Handler{service: service}
}
```

## 命令参考

### goon init

初始化新的 Gin Web 项目。

```bash
goon init <项目名称> [选项]
```

选项：
- `-m, --module <名称>`: 指定 Go module 名称（默认使用项目名称）
- `--minimal`: 只生成核心文件和目录

示例：
```bash
goon init myapp
goon init myapp --minimal
goon init myapp -m github.com/user/myapp
```

### goon add

添加新模块到项目。

```bash
goon add <模块名称> [选项]
```

选项：
- `-l, --layers <层列表>`: 指定要生成的层，逗号分隔（handler,service,model,repository,schema）
- `--register`: 是否自动注册路由（默认为 true）

示例：
```bash
goon add user
goon add product --layers=handler,service
goon add order --register=false
goon add userProfile  # 支持驼峰命名
```

### goon add pkg

添加功能包到项目（适用于 minimal 模式后按需添加）。

```bash
goon add pkg <包名称>
```

可用的功能包：
- `validator` - 数据验证
- `database` - 数据库连接
- `jwt` - JWT 认证
- `utils` - 通用工具
- `cache` - 缓存功能
- `email` - 邮件发送
- `upload` - 文件上传
- `pagination` - 分页工具
- `testutil` - 测试工具

示例：
```bash
goon add pkg cache
goon add pkg jwt
goon add pkg email
```

### goon list

列出项目中的所有模块。

```bash
goon list
```

显示 internal 目录下的所有业务模块及其包含的文件。

### goon list-pkg

列出所有可用的功能包。

```bash
goon list-pkg
```

显示可以通过 `goon add pkg` 添加的所有功能包。

### goon remove

删除项目中的模块。

```bash
goon remove <模块名称> [选项]
```

选项：
- `-f, --force`: 强制删除，不要求确认

示例：
```bash

goon remove product --force
```

注意：删除模块后需要手动从 router.go 中移除相关路由注册代码。

## 最佳实践

### 1. 使用 Minimal 模式开始

对于小型项目或原型开发，建议使用 `--minimal` 模式：

```bash
goon init myapp --minimal
```

这样可以避免生成不需要的功能包，保持项目简洁。

### 2. 按需生成层级

如果不需要完整的分层架构，可以只生成需要的层：

```bash
# 简单的 API，只需要 handler
goon add simple --layers=handler

# 需要业务逻辑但不需要数据库
goon add logic --layers=handler,service
```

### 3. 利用幂等性补充文件

如果后续需要添加缺失的层，可以再次运行命令：

```bash
# 初始只生成 handler
goon add user --layers=handler

# 后续添加 service 和 repository
goon add user --layers=service,repository
```

### 4. 统一命名风格

虽然工具支持多种命名格式，但建议在项目中保持统一的命名风格：

```bash
# 推荐：使用 snake_case
goon add user_profile
goon add order_history

# 或者使用 camelCase（会自动转换为 snake_case）
goon add userProfile
goon add orderHistory
```
